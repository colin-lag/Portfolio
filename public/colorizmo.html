ROBO COLORIZMO

OVERVIEW

October - December 2019

Collaborator: Alexandre Berkovic

The task of this project was to create a robotic contraption which a user could interact with. My partner and I decided to create a 3-axis robotic gripping arm that could be controlled by the user. When the controls and build were finished, we decided to create and implement a coloured cube sorting computer vision algorithm.

FABRICATION

The robotic arm was built from laser cut acrylic pieces, with 3D printed component and a metal base. Acrylic was chosen due to its ease for prototyping iterations using the laser cutter as well as its aesthetic appeal. Specific custom pieces such as a servo mount, or a timing belt tension clamp were 3D printed for rapid prototyping and iterating.

Due to the limited number of available pins on our Arduino microcontroller board, the servo controlling the vertical movement of the gripper was removed and instead a mechanical alternative was created.

CONTROLS

To control the robotic arm 4 potentiometers were used. These controlled the 3 servos and the stepper motor in the base. These allowed for intuitive control of the different components of the arm and enabled the user to switch from free flow control to cube detection.

On the controller is the button to toggle the cube detection script on and off, as well as the LED indicating the colour of the cube when it has identified it.

ROBOTIC CONTROL SCRIPT

The robotic control script was made on an Arduino Uno and coded in C/C++. The Arduino and Raspberry Pi are connected through USB and communicate through UART to transmit information. The transmitted data can be directions of rotation, a cue to stop the rotation or the identified colour of a cube.

COMPUTER VISION ALGORITHM

To find and sort the coloured cubes, I coded a Python algorithm using OpenCV running on a Raspberry Pi. The Python script would run in the background waiting for a button to be pressed on the controller, subsequently calling an interrupt and launching the computer vision script.

Resize frame

Transform to grayscale

Bilateral filtering

Canny edge detection

Shi-Tomasi feature detection

Centroid calculation

By using the low depth of field of the camera to our advantage and subsequent layers of image filtering, the algorithm would identify the centroid of the cube with a very high accuracy. When the algorithm determines that the cube is aligned with the gripper, it uses the RGB value of the centroid to identify the colour of the cube and reports this to the Arduino script. This in turn triggers the pre-programmed command to grab the cube and release it in a pre-defined location with the same colour cubes.

This project motivated to learn more about Computer Vision and finding way of applying it to future projects. I completed an online course over summer to learn about OpenCV in more depth. I then worked on several projects combining Deep Learning and Computer Vision with using datasets found on Kaggle.
